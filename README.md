🍔 Cardápio Online - Projeto Frontend
📌 Sobre o projeto

Este projeto é uma aplicação web de cardápio online para restaurante, com foco em experiência do usuário e interatividade.

A aplicação simula um sistema de delivery, permitindo navegação por categorias, visualização dinâmica de produtos e carrinho de compras com atualização em tempo real.

🛠️ Tecnologias utilizadas
🌐 HTML5 → estrutura semântica da aplicação
🎨 CSS3 → estilização, layout e responsividade
⚙️ JavaScript → lógica e interatividade
📦 jQuery → manipulação do DOM e eventos
🧩 Bootstrap → grid e responsividade
🎯 Font Awesome → ícones
✨ Animate.css → animações visuais
🎬 WOW.js → animações ao scroll
🧠 Estrutura do código
📄 HTML centralizado no index.html, organizando todas as seções do site
🎨 CSS dividido entre estilos principais e responsividade para mobile
⚙️ JavaScript separado entre lógica da aplicação e dados do cardápio

A aplicação utiliza manipulação direta do DOM para renderizar produtos dinamicamente a partir de um objeto de dados local, simulando o comportamento de uma API.

📦 Arquitetura de dados

Os produtos do cardápio são armazenados em um arquivo JavaScript (dados.js), organizados por categorias como burgers, pizzas, bebidas, sobremesas e outros.

Cada item contém informações como:

id
nome
descrição
imagem
preço
⚡ Funcionalidades
🧾 Listagem dinâmica de produtos por categoria
🔄 Paginação com carregamento incremental (“Ver mais”)
➕ Controle de quantidade por item
🛒 Carrinho de compras com atualização em tempo real
📱 Interface responsiva para mobile e desktop
🎞️ Animações para melhor experiência visual
💡 Observações técnicas

Projeto 100% front-end, sem backend ou banco de dados.

Toda a lógica de dados é local, baseada em objetos JavaScript, simulando integração com API.

Foco em organização de código, separação de responsabilidades e boas práticas de estrutura front-end.
